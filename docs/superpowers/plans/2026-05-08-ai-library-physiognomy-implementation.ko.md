# AI 관상가 고양이 구현 계획 한국어판

이 문서는 사용자가 읽기 쉽게 정리한 한국어 버전입니다. 실제 작업자가 그대로 따라 실행할 상세 코드/명령어 계획은 같은 폴더의 영문 문서 `2026-05-08-ai-library-physiognomy-implementation.md`에 있습니다.

## 목표

새 레포 `ai-library-physiognomy`에서 대학 도서관 B2B 부스용 MVP를 만든다.

학생은 이름, 학번, 성별, 생년월일, 선호 독서 카테고리를 입력하고 웹캠 얼굴 분석을 진행한다. 서비스는 얼굴 메트릭과 생년월일 기반의 가벼운 관상/사주 컨셉을 사용해 16가지 독서 처방 타입 중 하나를 정하고, 학생에게 지금 필요한 책 3권을 추천한다.

서비스명은 **AI 관상가 고양이**이며, 안경 쓰고 의사 가운을 입은 귀여운 고양이 마스코트가 시작/분석/결과/오류 화면에 등장한다.

## 전체 구조

- 프론트엔드: Next.js 14 App Router, TypeScript, Tailwind
- 얼굴 분석: 기존 AI Face Roasting 레포의 MediaPipe/웹캠 파이프라인 재사용
- AI 생성: Gemini
- 데이터 저장: 새 Supabase 프로젝트
- 배포: 새 Vercel 프로젝트
- 책 데이터: 도서관 정보나루 OpenAPI 수집 후 Gemini 태깅, Supabase import

추천 방식은 하이브리드다.

1. 앱이 얼굴 메트릭과 사용자 입력을 수집한다.
2. 서버가 Supabase 책 DB에서 후보 책 10~20권을 고른다.
3. Gemini가 16가지 독서 처방 타입 중 하나를 선택한다.
4. Gemini가 후보 책 안에서만 최종 3권을 고르고 추천 문구를 작성한다.
5. 서버는 DB에 존재하는 책만 결과로 저장한다.

## 작업 3. Supabase DB 스키마와 보관 정책 만들기

Supabase에 필요한 테이블과 Storage bucket을 만든다.

주요 테이블:

- `library_sessions`: 학생 입력, 얼굴 분석 결과, 추천 결과 저장
- `books`: 추천 가능한 책 DB
- `service_events`: 운영/오류 로그

보관 정책:

- 얼굴 이미지는 결과 화면에서 24시간까지만 표시
- 얼굴 이미지는 Storage에 최대 30일 보관 후 삭제
- 개인 결과는 30일간 재조회 가능
- 30일 이후 개인 식별 정보와 개별 결과 원문 삭제
- 장기 보관은 익명 집계 통계만 남김

검증:

- 24시간 후 얼굴 이미지 미노출 helper 테스트
- 30일 후 세션 만료 helper 테스트
- Supabase migration push

## 작업 4. 책 데이터 파이프라인 만들고 Supabase에 책 DB 준비하기

책 DB는 제가 직접 준비하는 범위다.

흐름:

1. 도서관 정보나루 OpenAPI에서 도서 데이터 수집
2. raw 데이터 저장
3. MVP용 책 데이터로 정규화
4. Gemini로 책마다 추천 태그 부여
5. MVP용 청구기호와 위치 더미 생성
6. Supabase `books` 테이블에 import
7. 가공된 seed 파일을 레포에 커밋

필요한 환경변수:

- `DATA4LIBRARY_AUTH_KEY`
- `GEMINI_API_KEY`
- `NEXT_PUBLIC_SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`

산출물:

- `data/books/books.raw.json`
- `data/books/books.normalized.json`
- `data/books/books.tagged.json`
- Supabase `books` 테이블에 업로드된 책 데이터

검증:

- 정규화 테스트
- API 수집 성공
- Gemini 태깅 성공
- Supabase import 성공

## 작업 7. Gemini 프롬프트/스키마와 분석 API 만들기

`/api/analyze`를 만든다.

API 역할:

1. 학생 입력, 얼굴 메트릭, 얼굴 이미지 수신
2. Supabase에 세션 생성
3. 얼굴 이미지 Storage 업로드

## 작업 8. 고양이 마스코트와 입력/분석 UI 만들기

마스코트 에셋 5종을 준비한다.

- `idle`: 시작 화면
- `diagnose`: 웹캠/분석 중
- `reading`: 책 후보 고르는 중
- `result`: 결과 화면
- `retry`: 오류/재시도

입력 화면 필드 순서:

1. 이름
2. 학번
3. 성별
4. 생년월일
5. 선호 독서 카테고리
6. 개인정보처리방침 및 이용약관 동의

화면:

- 시작/입력 화면
- 웹캠 분석 화면
- 분석 중 화면

검증:

- 마스코트 컴포넌트 접근성 테스트

## 구현 방식

사용자가 선택한 방식은 **서브에이전트 기반 구현**이다.

진행 원칙:

- 작업 단위별로 새 worker subagent를 붙인다.
- 각 작업이 끝나면 스펙 준수 리뷰를 한다.
- 그 다음 코드 품질 리뷰를 한다.
- 문제가 있으면 같은 작업자에게 수정시킨 뒤 다시 리뷰한다.
- 모든 작업이 끝날 때까지 중간에 불필요하게 멈추지 않는다.
