# AI 관상가 고양이 MVP 설계

작성일: 2026-05-08  
레포: `ai-library-physiognomy`  
상태: 사용자 리뷰 중, 구현 계획 작성 전

## 1. 목표

기존 AI 얼평 서비스를 그대로 수정하지 않고, 새 레포와 새 배포 프로젝트에서 대학 중앙도서관 B2B 입점용 MVP를 만든다.

제품명은 "AI 관상가 고양이"다. 학생이 부스형 PC에서 이름, 학번, 성별, 생년월일, 선호 독서 카테고리를 입력하고 웹캠 얼굴 분석을 진행하면, 안경을 쓰고 의사 가운을 입은 귀여운 고양이 관상가가 관상/사주 컨셉으로 현재 학생에게 필요한 책 3권을 처방해주는 체험이다.

핵심 목표는 단순한 "오늘의 책" 추천이 아니라, 사용자가 "이건 나한테 필요한 책 같다. 빌려봐야겠다"라고 느끼는 동기 부여형 독서 큐레이션이다. 문구 톤은 유머러스하고 MZ스럽게 가되, 추천 이유는 책을 실제로 빌리고 싶게 만들 만큼 설득력 있게 작성한다.

## 2. 제품 포지셔닝

서비스명은 "AI 관상가 고양이"다. 서비스 컨셉은 "고양이 관상가가 얼굴과 생년월일을 보고 지금 나에게 필요한 책을 처방해준다"이다.

표면 경험은 귀엽고 재미있지만, 결과는 학생 개인에게 맞춘 처방전처럼 느껴져야 한다. 관상과 사주는 과학적 진단이나 성격 단정이 아니라 엔터테인먼트형 독서 큐레이션 장치로 사용한다.

카피는 고정 문구를 많이 넣기보다 Gemini가 분석 결과에 맞춰 생성한다. 프롬프트에는 좋은 톤 예시, 금지선, 추천 동기 부여 목적을 명확히 넣는다.

카피 가드레일:

- 외모 비하 금지
- 운명 단정 금지
- 성격/능력/정신건강 낙인 금지
- 법적/사회적으로 민감한 개인정보나 고위험 속성의 추론 금지
- 책 추천 이유는 사용자 입력, 관상/사주 키워드, 책의 주제를 연결
- 결과는 재미있게 시작하되 실제 행동으로 이어지게 마무리

"민감한 특성"은 일반적인 성향 농담을 막자는 뜻이 아니다. 결과 카피는 "생각이 많은 타입", "관계 정리가 필요한 타입", "실행 버튼을 눌러야 하는 타입"처럼 독서 큐레이션에 필요한 성향 표현을 적극적으로 사용할 수 있다.

금지할 것은 얼굴/생년월일만으로 정치 성향, 종교/신념, 건강 상태, 정신질환, 성생활/성적 지향, 유전 정보, 범죄 성향, 인종/민족, 가정 형편, 소득 수준처럼 민감하거나 피해가 큰 속성을 단정하는 것이다. B2B 대학 도서관 서비스에서는 재미보다 신뢰 리스크가 더 큰 영역이므로 프롬프트에서 명시적으로 막는다.

## 3. 고양이 마스코트

마스코트는 안경을 쓰고 의사 가운을 입은 귀여운 고양이다. 역할은 "진단하는 AI"를 차갑게 보이지 않게 만들고, 관상/사주라는 소재를 안전하고 장난스러운 독서 처방 경험으로 감싸는 것이다.

마스코트 사용 원칙:

- 앱 전체에 과하게 반복하지 않고 주요 전환점에 등장시킨다.
- 시작 화면, 분석 중 화면, 결과 헤드라인, 재조회 빈 상태, 오류/재시도 안내에 사용한다.
- 결과 책 카드마다 넣지는 않는다. 책 추천의 설득력을 방해하지 않게 한다.
- 기존 디자인 시스템의 컬러, 카드 스타일, 타이포와 충돌하지 않게 작은 companion asset으로 쓴다.

구현 제안:

- 런타임에 이미지를 생성하지 않고, 사전에 생성한 투명 배경 WebP/PNG 에셋을 `public/mascot/`에 둔다.
- 초기 에셋은 5종으로 제한한다.
  - `idle`: 시작 화면
  - `diagnose`: 웹캠/분석 중
  - `reading`: 책 후보를 고르는 중
  - `result`: 결과 헤드라인
  - `retry`: 오류/재시도
- React 컴포넌트는 `Mascot` 또는 `MascotGuide` 하나로 감싼다. `variant`, `size`, `message`만 props로 받게 해서 화면마다 일관되게 쓴다.
- 첫 구현은 정적 이미지로 충분하다. 이후 B2B 데모 완성도를 올릴 때 눈 깜빡임/꼬리 흔들림 정도의 Lottie 또는 CSS 애니메이션을 추가한다.
- 이미지 생성 프롬프트는 "cute Korean web mascot, cat wearing round glasses and white doctor's coat, warm library mood, transparent background, clean app icon style, not too childish" 방향으로 잡는다.

## 4. MVP 범위

포함한다:

- 행사 모드용 시작, 입력, 웹캠, 분석 중, 결과 플로우
- 기존 얼굴 캡처, MediaPipe, Gemini, Supabase 구조의 재사용
- 하이브리드 추천 로직
- 국립중앙도서관 등 공공 데이터 기반 책 시드 DB
- 책 태그, 카테고리, 청구기호, 위치 더미 데이터
- 결과 저장 및 30일 재조회
- 얼굴 이미지 24시간 후 미노출, 30일 후 삭제
- 기존 디자인 시스템을 유지한 입력/결과/어드민 화면 개편
- AI 관상가 고양이 마스코트 에셋과 UI 컴포넌트
- 최소 범위의 어드민 KPI 화면

제외한다:

- 실제 학교 OPAC 연동
- 학교 SSO/학번 검증
- 익명 참여
- 영어 UI
- QR로 모바일 전송
- 실제 대출 전환 추적
- 상시 모드
- 정교한 책 위치 지도

## 5. 사용자 입력

입력 화면 필드 순서는 고정한다.

1. 이름
2. 학번
3. 성별
4. 생년월일
5. 선호 독서 카테고리
6. 개인정보처리방침 및 이용약관 동의

MVP는 실명 참여만 지원한다. 익명 참여는 제외한다.

태어난 시간은 받지 않는다. 사주 요소는 년주/월주 수준의 가벼운 컨셉으로 사용한다.

## 6. 이름 개인화와 한국어 조사

결과 화면은 사용자가 입력한 이름을 여러 번 사용한다. 단, 한국 이름은 기본적으로 성을 제거한 이름만 사용한다.

예:

- `박영민` -> `영민`
- `박건우` -> `건우`

호칭과 조사는 이름의 마지막 글자 받침 여부에 따라 자연스럽게 처리한다.

예:

- `영민` -> `영민이에게`, `영민아`
- `건우` -> `건우에게`, `건우야`

앱 쪽 helper에서 반복 UI 문구에 필요한 조사를 안정적으로 보정한다. Gemini 프롬프트에도 한국어 호칭과 조사를 자연스럽게 처리하라는 지침을 넣지만, 핵심 UI 문구는 앱에서 제어한다.

우선 처리할 조사:

- 이/가
- 은/는
- 을/를
- 아/야
- 이에게/에게
- 과/와
- 으로/로

## 7. 추천 아키텍처

추천은 하이브리드 구조로 만든다.

1. 사용자는 책을 직접 고르지 않는다. 유저 입력은 이름, 학번, 성별, 생년월일, 선호 독서 카테고리뿐이다.
2. 얼굴 분석과 입력값으로 1차 관상/사주 키워드를 만든다.
3. 서버가 내부 책 DB에서 선호 카테고리와 태그 기준으로 추천 가능한 책 후보 10~20권을 자동 선정한다.
4. Gemini는 이 내부 후보 안에서만 최종 3권을 선택하고 추천 이유를 생성한다.
5. 최종 결과에는 DB에 존재하는 책의 제목, 저자, 청구기호, 위치만 표시한다.

초기 태그 예:

- 집중력 회복
- 관계 정리
- 진로 탐색
- 자기확신
- 실행력
- 감정 리셋
- 사고 확장
- 문해력
- 회복
- 커리어

핵심 제약은 LLM이 없는 책을 지어내지 못하게 하는 것이다. 책 선택은 DB 후보 안에서만 이뤄지고, Gemini는 해석과 설득 문구 생성에 집중한다.

## 8. 16가지 독서 처방 타입

메인 헤드라인은 자유 문장만 쓰지 않고, 관상/사주 분석 기반의 16가지 독서 처방 타입 중 하나를 먼저 보여준다. 타입은 결과의 첫 인상을 만들고, 사용자가 친구에게 공유하기 쉬운 이름표 역할을 한다.

타입 산출 방식:

1. Gemini가 얼굴 메트릭, 생년월일 기반 가벼운 사주 키워드, 선호 카테고리, 책 후보 태그를 보고 아래 16개 enum 중 하나를 선택한다.
2. 서버는 응답 JSON의 `reading_type.code`가 허용된 enum인지 검증한다.
3. 결과 화면의 메인 헤드라인은 `reading_type.display_name`과 이름 호칭을 결합해 보여준다.
4. 타입은 재미있는 큐레이션 장치이며, 성격/능력/건강에 대한 단정으로 보이지 않게 설명한다.

초기 16가지 타입:

| Code | 표시 타입 | 헤드라인 예시 | 추천 방향 |
|---|---|---|---|
| `focus_reboot` | 집중력 리부트형 | `영민아, 지금은 집중력 리부트 처방이 떴다` | 집중, 습관, 몰입 |
| `thought_overload` | 생각 과부하 정리형 | `영민이의 머릿속 탭 47개, 이제 정리할 시간` | 사고 정리, 글쓰기, 철학 입문 |
| `career_compass` | 진로 나침반형 | `영민이에게 필요한 건 스펙보다 방향 감각` | 진로, 커리어, 자기이해 |
| `action_button` | 실행 버튼 수리형 | `영민아, 계획 말고 실행 버튼부터 고치자` | 실행력, 생산성, 행동 |
| `emotion_reset` | 감정 리셋형 | `영민이의 마음 배터리, 지금 충전 필요` | 감정 회복, 에세이, 심리 교양 |
| `relationship_translator` | 인간관계 번역기형 | `영민이에게 필요한 건 사람 마음 자막` | 관계, 대화, 사회심리 |
| `self_trust` | 자기확신 보강형 | `영민아, 남 눈치 말고 내 기준부터 세우자` | 자기확신, 삶의 기준, 에세이 |
| `ambition_strategy` | 야망 전략가형 | `영민이의 야망은 있는데 지도 업데이트가 필요함` | 전략, 경영, 커리어 |
| `rest_prescription` | 휴식 처방형 | `영민아, 쉬는 것도 과제처럼 해내야 할 때` | 휴식, 번아웃 예방, 회복 |
| `curiosity_explorer` | 취향 발굴 탐험형 | `영민이의 취향 레이더, 아직 숨은 보물 찾는 중` | 교양, 입문서, 인문 |
| `reality_tuning` | 현실 감각 튜닝형 | `영민이에게 필요한 건 낭만과 현실의 밸런스` | 경제, 사회, 실용 |
| `creativity_walk` | 창의력 산책형 | `영민아, 아이디어는 책장 사이에서 튀어나온다` | 창의성, 예술, 문학 |
| `language_muscle` | 문해력 근육형 | `영민이의 생각 근육, 오늘은 문장으로 키우자` | 문해력, 고전, 글쓰기 |
| `worldview_expand` | 세계관 확장형 | `영민아, 지금 필요한 건 뇌 속 지도 확장팩` | 과학, 역사, 사회, 철학 |
| `confidence_softener` | 마음 말랑 회복형 | `영민이의 표정엔 괜찮은 척이 살짝 보임` | 위로, 문학, 자기돌봄 |
| `deep_dive_scholar` | 딥다이브 학자형 | `영민아, 얕게 많이 말고 하나를 깊게 팔 타이밍` | 전문 교양, 연구, 심화 독서 |

Gemini 응답에는 타입 코드, 표시 타입, 메인 헤드라인, 타입 설명, 추천 책 3권이 포함되어야 한다. 메인 헤드라인은 타입명을 중심으로 하되 이름과 조사 규칙을 자연스럽게 반영한다.

## 9. 화면 흐름

MVP는 행사 부스용 90초 체험으로 설계한다.

### 시작 화면

- "AI 관상가 고양이" 컨셉
- 안경과 의사 가운을 착용한 고양이 마스코트
- 행사 부스용 짧은 CTA
- 체험 시작 진입

### 입력 화면

- 이름
- 학번
- 성별
- 생년월일
- 선호 독서 카테고리
- 개인정보처리방침 및 이용약관 동의

### 웹캠 분석 화면

- 기존 MediaPipe 캡처 흐름 재사용
- 얼굴 인식 안정화 후 자동 캡처
- 실패 시 재시도 안내

### 분석 중 화면

- Gemini 호출 중 대기
- 관상/사주 해석과 책 매칭이 진행 중이라는 느낌의 짧은 문구
- `diagnose` 또는 `reading` 마스코트 표시
- 전체 체험 90초 내 유지

### 결과 화면

기존 AI 얼평 결과 화면처럼 분석 결과가 먼저 나오고, 책 추천은 마지막에 나온다.

순서:

1. 16가지 독서 처방 타입 기반 메인 헤드라인
2. 얼굴 이미지
3. 얼굴/관상 분석 결과
4. 사주 기반 가벼운 해석
5. 독서 니즈 요약
6. 추천 책 3권

메인 헤드라인과 추천 섹션에는 성을 뺀 이름을 사용한다.

예: `박영민` 입력 시 `영민아, 지금은 집중력 리부트 처방이 떴다`

책 카드에는 다음을 표시한다:

- 제목
- 저자
- 왜 지금 필요한지
- 청구기호
- 도서관 위치
- 행동 유도 문구

### 재조회 화면

- 학번 + 생년월일로 조회
- 30일 내 결과 재조회 가능
- 생성 후 24시간이 지난 경우 얼굴 이미지는 표시하지 않음
- 이름 개인화 결과와 추천 책은 계속 표시

## 10. 데이터 보관과 개인정보

MVP는 명시적 동의를 받은 뒤 이름, 학번, 성별, 생년월일, 선호 독서 카테고리, 얼굴 이미지, 분석 결과, 추천 결과를 저장한다.

보관 정책:

- 얼굴 이미지는 결과 화면에서 생성 후 24시간까지만 표시한다.
- 얼굴 이미지는 서비스 품질 관리 목적에 한해 Supabase Storage에 최대 30일 보관 후 삭제한다.
- 개인 결과와 추천 기록은 30일간 보관한다.
- 학생은 학번 + 생년월일로 30일 내 결과를 재조회할 수 있다.
- 30일 이후 개인 식별 정보, 얼굴 이미지, 개별 결과 원문은 삭제한다.
- 장기 보관 데이터는 개인을 재식별할 수 없는 집계 통계만 남긴다.
- 익명 통계는 장기 보관한다.

어드민 정책:

- 어드민은 운영 통계와 추천 결과를 볼 수 있다.
- 얼굴 이미지는 기본적으로 어드민 화면에서 직접 열람하지 않는다.
- 학번은 어드민 화면에서 마스킹한다.

동의 문구에는 얼굴 이미지가 24시간 후 화면에서 보이지 않으며, 품질 관리 목적에 한해 최대 30일 보관 후 삭제된다는 점을 명확히 고지한다.

## 11. 책 데이터와 위치 안내

MVP에서는 실제 학교 OPAC 연동 없이 공공 도서 데이터와 시뮬레이션 위치 데이터로 간다.

- 국립중앙도서관 등 공공 데이터에서 카테고리별 책 데이터를 가져와 초기 DB를 만든다.
- 책마다 내부 태그를 붙인다.
- 각 책에는 MVP용 청구기호와 위치를 넣는다.
- 실계약 시 학교 OPAC/소장자료 API로 교체할 수 있게 `BookProvider`와 `LocationProvider` 어댑터 구조를 둔다.

결과 화면에는 학생이 바로 움직일 수 있도록 책 제목, 저자, 추천 이유, 청구기호, 위치, 행동 유도 문구를 보여준다.

## 12. 운영/어드민

기존 `admindata` 패턴을 재활용하되 MVP에는 아래만 담는다.

- 오늘 참여자 수
- 오늘 추천된 책 수
- 선호 독서 카테고리 분포
- 추천된 책 리스트
- 추천 분야 / 태그 분포
- 세션 목록

오늘 참여자 수는 숫자 카드와 차트로 모두 보여준다.

디자인 시스템은 기존 앱 스타일을 유지한다. 새 관리자 디자인 언어를 만들지 않고, 기존 컬러, 타이포, 카드/섹션 스타일, 레이아웃 감각을 최대한 유지한다.

## 13. B2B 가치 제안

메인 KPI는 도서관 방문자 수 증가다.

보조 KPI:

- 추천 도서 대출/열람 전환
- 학생 참여 데이터 리포트

영업 메시지:

> 학생을 도서관으로 오게 만드는 체험형 큐레이션 부스.

MVP에서는 실제 대출 전환까지 추적하지 않고, 방문/참여/추천 노출 지표를 먼저 잡는다. 대출 전환은 실계약 후 OPAC 연동 단계에서 확장한다.

## 15. 검증 기준

- 학생 한 명이 90초 이내에 결과까지 도달한다.
- Gemini가 DB에 없는 책을 추천하지 않는다.
- 결과 화면이 이름을 자연스럽게 반복 사용한다.
- 받침 기반 조사/호칭이 어색하지 않다.
- 결과 화면의 메인 헤드라인은 16가지 독서 처방 타입 중 하나를 기반으로 한다.
- 타입 코드는 서버에서 enum 검증을 통과한다.
- 고양이 마스코트는 시작/분석/결과/오류 상태에서 기존 디자인 시스템과 어긋나지 않게 표시된다.
- 재조회는 30일 내 가능하다.
- 생성 24시간 후 재조회에서는 얼굴 이미지가 보이지 않는다.
- Supabase Storage의 얼굴 이미지는 30일 후 삭제된다.
- 어드민에서 오늘 참여자 수와 추천 책/태그 분포를 볼 수 있다.
- 오늘 참여자 수는 카드와 차트로 모두 보인다.
- 기존 디자인 시스템과 크게 어긋나지 않는다.
